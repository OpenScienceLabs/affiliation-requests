# Maintainer Guide

This guide explains what OSL expects from maintainers of affiliated projects.

## Maintainer responsibilities

Maintainers remain responsible for their own project. Affiliation does not
transfer ownership, governance, intellectual property, roadmap control,
maintenance responsibility, releases, support, or community moderation to OSL.

Affiliated project maintainers should:

- keep an OSI-approved license and public license file in place;
- keep a Code of Conduct and reporting path in place;
- keep maintainers reachable through the documented contact path;
- keep repository, website, and documentation links accurate;
- respond to maintenance review pings;
- keep the OSL acknowledgement accurate in README/docs;
- notify OSL if the project is archived, moved, renamed, transferred,
  relicensed, paused, or no longer wants affiliation; and
- avoid language that implies OSL ownership, certification, sponsorship,
  guarantees, or control.

## Acknowledgement

Accepted projects should add the acknowledgement described in
[acknowledgement.md](acknowledgement.md). If affiliation is paused, removed, or
changed, update or remove the acknowledgement.

## Maintenance reviews

OSL may periodically check public metadata such as repository visibility,
archived status, license, Code of Conduct, README, security reporting path,
activity, and broken links. Automation supports this process, but humans make
final decisions about removal or reactivation.

If you receive a maintenance review issue, please respond with current links,
status, and any context that reviewers should know.

## GSoC, internships, grants, and similar programs

Affiliation does not guarantee access to GSoC, internships, grants, sponsorship,
contributors, mentors, funding, promotion, or event participation.

If you want to propose ideas for a limited program, prepare:

- mentors with enough time to support contributors;
- scoped project ideas;
- public issues or milestones;
- contribution and development setup docs;
- communication channels;
- contributor review capacity; and
- realistic expectations about external selection and limited slots.

Even when OSL participates in a program, not every affiliated project idea can be
selected or funded.
